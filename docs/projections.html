Northwood — Aggregate Projections

Companion to event-flow.html, re-pivoted by aggregate rather than event. For each Aggregate Root, this view answers: "which downstream services hold a cached / projected / saga-managed view of me, and in which tables?"

Tree structure: Source Service → Aggregate Root → Destination Service → Projection Table. Each non-leaf node is collapsible — click the ▶ marker. Counts in parentheses are the size of the immediate children.

Each leaf row tags its write target with a kind chip:

  • read-model — classic denormalised projection. Lives in a consumer schema, fed by inbox handlers, no consumer-side command writes (e.g. sales.product_card, reporting.sales_order_360_view, finance.product_accounting).
  • saga — saga state row mutated by inbox handlers reacting to ack-style events from cooperating services (e.g. sales.sales_order_fulfilment_saga, manufacturing.make_to_order_saga).
  • aggregate — destination service's own aggregate, mutated through commands triggered by inbox handlers (e.g. inventory.stock_reservation on StockReservationRequested, finance.journal_entry_header on ShipmentPosted).
